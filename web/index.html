06 // Concepts

Key DevOps Concepts

Docker & Containerization
Packages an app and all its dependencies into an isolated container — runs identically everywhere. Each service (frontend, backend, DB, translator) gets its own Dockerfile with a base image, dependencies, and startup command.
Docker Compose
Defines and runs multi-container apps with a single YAML file. Services share a network (appnet), environment variables, volumes, and health checks — all orchestrated locally with one command.
Kubernetes (K8s)
Container orchestration — automatically schedules, scales, and heals containers. Key objects: Deployment (stateless apps), StatefulSet (databases), Service (networking), Ingress (external access), PVC (persistent storage).
Helm Charts
The package manager for Kubernetes. Converts static YAML manifests into dynamic templates with values.yaml. Enables versioned releases, one-command upgrades, and instant rollback with helm rollback.
CI/CD — GitHub Actions
Continuous Integration & Deployment: every push triggers automated tests → Docker build → integration tests → push to registry → deploy to production. Zero manual steps from commit to live.
StatefulSet vs Deployment
Deployment is for stateless apps (any pod is interchangeable). StatefulSet is for databases — each pod has a stable identity (postgres-0) and its own persistent volume that survives restarts.
Ingress & Load Balancer
Ingress is an L7 HTTP router inside the cluster — routes requests by path or hostname to the right service. The Nginx Ingress Controller exposes the app at translator.local and handles path-based routing.
AWS S3 + CloudFront + OAC
S3 stores static frontend files. CloudFront (CDN) serves them globally with caching. OAC (Origin Access Control) makes S3 fully private — only CloudFront can read it, blocking direct access.
EC2 + Docker on Cloud
EC2 is a virtual Linux server on AWS. The backend runs inside a Docker container on EC2 — GitHub Actions SSHes in on every push, pulls the latest image, and restarts the container automatically.
OIDC Authentication
OpenID Connect lets GitHub Actions authenticate to AWS without storing static access keys. GitHub generates a short-lived token per job — dramatically more secure than long-lived credentials.
Sealed Secrets
Encrypts Kubernetes Secrets so they can be safely committed to Git. The kubeseal tool encrypts with the cluster's public key — only the cluster can decrypt. Solves the "secrets in git" problem.
Liveness & Readiness Probes
Kubernetes health checks: Readiness — is the pod ready to receive traffic? Liveness — is the pod still alive? If liveness fails, K8s restarts the pod automatically. Both hit the /health endpoint.
